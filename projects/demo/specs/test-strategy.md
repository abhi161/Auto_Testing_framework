## 2. Authentication

| Attribute           | Detail                                                                  |
|---------------------|-------------------------------------------------------------------------|
| **Type**            | Simple HTML form — **not** Azure AD / Cognito federation               |
| **Login route**     | `GET /login`                                                            |
| **Username field**  | `#username`                                                             |
| **Password field**  | `#password`                                                             |
| **Submit button**   | `button[type="submit"]`                                                 |
| **Success signal**  | `.flash.success` ∋ `"You logged into a secure area"` + URL → `/secure` |
| **Failure signal**  | `.flash.error` ∋ `"Your username is invalid"`, stays on `/login`       |
| **Logout signal**   | `.flash.success` ∋ `"You logged out of the secure area"`, URL → `/login` |

> **Note:** `tests/helpers/auth.helper.ts` (Azure AD scaffold) is present as a framework requirement
> but is **not invoked** for this project. Direct form selectors are used throughout.

**Login flow (canonical step-by-step):**
```
1. page.goto('/login')
2. page.locator('#username').fill(process.env.TEST_USER)
3. page.locator('#password').fill(process.env.TEST_PASS)
4. page.locator('button[type="submit"]').click()
5. Expect: URL contains /secure  AND  .flash.success is visible
```

**Session reuse:** For tests that require a logged-in state (TC-DEMO-008, TC-DEMO-040),
use `beforeAll` + `storageState` to log in once per `describe` block. Never re-authenticate per test.

## 8. Handoff Notes for Test Planner

1. **CSV is authoritative.** `docs/actualtestcases.csv` has 40 rows — all must be mapped 1-to-1 to
   spec test cases. Do not add, remove, or rename test case IDs without updating the CSV.
2. **All spec files exist** under `tests/`. The Planner should validate each spec against its CSV rows,
   confirm selectors match `specs/selectors.md`, and flag any gaps or mismatches.
3. **Session reuse** is mandatory for auth-required tests (TC-DEMO-008, TC-DEMO-040):
   use `beforeAll` + `page.context().storageState()` — never re-login per test.
4. **No Azure AD flow.** The `auth.helper.ts` scaffold is present but must not be invoked here.
5. **Issues register** (`docs/issues.csv`) is currently empty — the Planner/Executor should populate
   it if any test reveals a defect (real app bug or design deviation).
6. **Viewport** must be `1920×1080` with `deviceScaleFactor: 1` for all tests.
7. **No arbitrary `waitForTimeout`.** Use concrete waits (`waitForSelector`, `waitForLoadState`,
   `waitForResponse`, etc.) per `playwright-rules`.
8. **Traceability:** every `test(...)` block must carry a `// TC-DEMO-XXX` comment mapping it to the
   CSV row and the applicable `BR-###` (where present).
